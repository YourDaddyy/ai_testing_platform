# Project Rules (Persistent Context)
The current AI session must read and obey these rules immediately upon initialization.

## 1. Diagnostic / Scratch Scripts Rule
- **NEVER** create temporary testing, diagnostic, or scratch scripts in the project root directory.
- ALL test scripts (like `test_bssp.js`, `test.js`, test `cli.py`, etc.) MUST be written to an OS temporary directory (like `/tmp/` or `%TEMP%`), or to a dedicated `.gemini/` or `tests/scratch/` directory.
- If the script requires project modules (like `node-ssh` in `node_modules`), create it carefully in an ignored directory (`.temp_scripts/` etc.) and ensure it is cleaned up, or explicitly use `import`/`require` absolute paths from the tmp directory. Do NOT pollute the root directory.

## 2. General Cleanliness
- Before declaring a task finished, ALWAYS clean up any temporary files or output files generated by the AI (such as `.json` dumps from scripts).
